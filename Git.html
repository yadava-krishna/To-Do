Git Commands Guide

1. Initialize a New Repository

git init

Initializes a new Git repository in the current directory, allowing you to start tracking changes in your project.

2. Check the Status of Your Changes

git status

Shows the current state of your working directory and staging area, listing any modified, untracked, or staged files.

3. Stage Files for Commit

git add .

Adds all changed files in the current directory to the staging area, preparing them to be committed.

git add filename

Stages a specific file for commit. Replace filename with the actual file name.

4. Commit Changes

git commit -m "Your commit message here"

Records the staged changes in the local repository with a message describing the changes.

5. Rename the Branch

git branch -M main

Renames the current branch to main. This is helpful if you started on a branch like master and want to switch to the main naming convention.

6. Add a Remote Repository

git remote add origin https://github.com/your-username/repository-name.git

Links the local repository to a remote GitHub repository, allowing you to push and pull changes to and from GitHub. Replace your-username/repository-name with your actual GitHub details.

7. Push Changes to Remote Repository

git push -u origin main

Pushes commits from the local repository to the remote repository on the specified branch (main here) and sets up tracking so that git push can be used without specifying origin and main every time.

git push -u origin master

Pushes to the master branch if the branch is named master instead of main.

8. Pull Remote Changes into Local Repository

git pull origin main --allow-unrelated-histories

Pulls changes from the remote repository’s main branch into the local repository, merging any differences. The --allow-unrelated-histories flag is used if the local and remote repositories don’t share a history, such as when they were initialized separately.

9. Force Push to Overwrite Remote Changes (Use with Caution)

git push -u origin main --force

Forcefully pushes your local changes to the remote repository, overwriting any changes in the remote branch. Use this only if you’re sure you want to overwrite the remote content.

10. Stage and Commit After Resolving Conflicts

git add filename

Stages the file(s) after resolving any merge conflicts, marking them as resolved.

git commit -m "Resolve merge conflicts"

Commits the changes after resolving conflicts, with a message indicating that conflicts have been resolved.

11. View Branches

git branch

Lists all branches in the repository, highlighting the current branch.
